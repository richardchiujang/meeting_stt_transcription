# ✅ WASAPI Loopback：你現在用的「最乾淨、最正統」方案（完整說明）

## 一、核心概念先講清楚（很重要）

你現在採用的是這個架構：

    Speaker (Loopback) ─┐
                        ├─> 你的 Python 程式 → STT
    Mic (Real)         ─┘

**這跟 Stereo Mix 是本質上的不同**。

### Stereo Mix 是什麼？

*   聲卡驅動提供的「虛擬麥克風」
*   一次只能選一個輸入
*   常被 OEM（Lenovo / Realtek）閹割或限制

### ✅ WASAPI Loopback 是什麼？

*   **Windows 核心音訊 API**
*   直接從「某一個輸出裝置（喇叭）」抓 **數位播放串流**
*   不走麥克風、不受 Realtek Stereo Mix 限制
*   **Windows 官方支援的機制**

Microsoft 官方定義也很清楚：

> WASAPI loopback 可捕捉正在「render（播放）」到某個輸出裝置的音訊串流    [\[youtube.com\]](https://www.youtube.com/watch?v=Ev5d0j-BwVk)

***

## 二、為什麼「你的舊版 WASAPI loopback 會覺得很爛？」

這不是你錯，是 **90% 人第一次踩 WASAPI loopback 都會中的雷**。

### ⚠️ 常見爛掉原因（你大概踩到 2～3 個）

#### 1️⃣ 一開始就用 16k 抓系統音（非常致命）

*   系統播放音通常是 **48k / 44.1k / stereo**
*   你直接用 16k/mono 去抓  
    → Windows 先幫你做一次「即時重新取樣 + 混音」  
    → 聲音立刻糊掉

✅ **正解**

*   Loopback：用 **輸出裝置 native format（通常 48k / stereo）**
*   再 **離線 downsample** 給 Whisper（16k / mono）

Audacity / WASAPI 文件也明確說：

> 要避免品質問題，loopback 應符合裝置取樣率    [\[uptogo.com.tw\]](https://uptogo.com.tw/%e8%b2%a1%e7%b6%93/%e8%ad%89%e5%88%b8%e5%95%86/%e5%b0%8f%e6%91%a9%e6%98%af%e5%93%aa%e4%b8%80%e5%ae%b6%ef%bc%9f/)

***

#### 2️⃣ 不小心抓到「被語音 DSP 處理的路徑」

你的機器有：

*   Intel Smart Sound
*   麥克風陣列
*   AEC / AGC / Noise Suppression

這些 **都是為「人聲通訊」設計的 DSP pipeline**。  
如果你把系統音跟麥克風混在一起、又讓 AGC 啟動：

*   聲音忽大忽小
*   影片聲被壓
*   Whisper 辨識率暴跌

Microsoft 甚至有專門的 AECMicArray 範例，說明這條 DSP 路徑的行為    [\[sohu.com\]](https://www.sohu.com/a/543470836_120607236)

✅ **正解**

*   Loopback 抓系統音（乾淨）
*   麥克風單獨抓（可選）
*   **不要一開始就混成一條**

***

#### 3️⃣ 用的 Python 套件「沒有真的開到 WASAPI loopback」

很多人第一版用的是：

*   `sounddevice`

但在 Windows 上：

*   **並不是每個 PortAudio build 都有 loopback**
*   很容易「你以為在抓 loopback，其實沒有」

這也是為什麼你後來會覺得：

> 「效果似乎很爛？我不確定！」

## 四、你現在「正確的工程心智模型」（先記這個）

> **WASAPI loopback 是「抓播放的數位資料」  
> 麥克風是「抓空氣進來的聲音」  
> 這兩條應該先分開，再決定要不要混**

## 五、建議你現在採用的「標準流程」（不寫程式也能理解）

### ✅ Step 1：只驗證系統音品質（關鍵）

*   用 WASAPI loopback
*   48k / stereo
*   錄 5～10 秒 YouTube

👉 這一步 **如果不乾淨，後面混什麼都沒用**

***

### ✅ Step 2：只驗證麥克風（人聲）

*   單錄 Mic
*   看有沒有被 AGC / AEC 搞爛（忽大忽小）

***

### ✅ Step 3：STT 時「先不混音」

實務經驗（而且很重要）：

> **Whisper 對「乾淨單一來源」的辨識率，  
> 遠勝過「兩個來源硬混」**

常見做法是：

*   系統音 → 跑一次 STT
*   麥克風 → 跑一次 STT
*   最後時間軸合併

***

## 六、為什麼這條路「對 ThinkPad / vPro 特別重要」

因為：

*   Stereo Mix 被 Lenovo 半封死 ✅
*   麥克風有企業級 DSP（AEC/NS/AGC）✅
*   WASAPI loopback 是 **Windows OS 級別** ✅

Microsoft 官方也說：

> Loopback 是 OS audio engine 的功能，不依賴硬體是否提供 loopback pin    [\[youtube.com\]](https://www.youtube.com/watch?v=Ev5d0j-BwVk)
